# /learning-journey — 设计用户完整学习旅程

从用户第一次接触到毕业就业的完整旅程设计，整合所有平台资源。

## 使用方法
```
/learning-journey [bootcamp目录]
```
例如：`/learning-journey ai-adoption-bootcamp`

## 执行步骤

1. **梳理用户旅程阶段**：
   ```
   发现 → 体验 → 注册 → 学习 → 毕业 → 就业/转型
   ```

2. **每个阶段整合平台资源**：

   **发现阶段**：
   - `/learn/ai-adoption` 免费章节（引流）
   - Slide Deck（营销演示）
   - 社媒内容（TikTok/LinkedIn/小红书）
   - Free Resources 页面的免费 AI 工具/认证推荐

   **体验阶段**：
   - 免费 Prompt Lab 体验（hello-ai 等）
   - AI 学习方向页面浏览
   - 免费 Roadmap 查看
   - Cheat Sheets 下载

   **学习阶段**：
   - 直播课 (Lesson type)
   - 录播视频 (Video type) — 独立 lesson，不是附属品
   - 图文教程 (Info type) — 独立 lesson
   - Prompt Lab 互动练习
   - Vibe Coding Lab
   - OpenClaw 章节
   - AI Tutor 一对一带练
   - Wiki 扩展阅读
   - 作业 + AI 自动评分 + 老师评审

   **毕业阶段**：
   - Capstone Project（毕业方案）
   - Demo Day（展示 + 评审）
   - Peer Review（互审能力）
   - 证书颁发

   **就业/转型阶段（P3 职业孵化器）**：
   - Path A: 内部转型（在现有公司跑真实 AI Pilot）
   - Path B: 外部咨询（以 JR 顾问身份接客户项目）
   - JobPin AI 简历优化
   - Mock Interview 模拟面试
   - Career Planning AI 职业规划
   - Job Referral 工作内推
   - Alumni 社群持续连接

3. **检查资源缺口**：
   - 每个阶段是否有足够的资源支撑？
   - 用户在哪个阶段容易流失？怎么防？
   - 哪些资源已有？哪些需要新建？

4. **输出**：
   - 用户旅程图（每阶段 → 用什么资源 → 用户的感受/目标）
   - 资源缺口清单
   - 流失风险点 + 应对策略
   - 可以输出为 HTML（用于 curriculum 站点展示）
